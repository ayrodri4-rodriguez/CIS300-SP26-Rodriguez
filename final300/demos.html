About

The above is a project conducted between myself and 5 other teammates. The goal was to essentially recreate the functionality of chat services such as EdDisscussion. In this 4 month long project, we held various weekly team meeting and discussed various different methods of testing, organization, and software architecture. The top features focues in the system follow CRUD functionality. They were as follows:

  • Staff members should be able to send private messages to both students and staff. This is to encourage a genial environment with all members
  • Staff members should be able to deploy CRUD (create, read, update, and delete) for supporting discussions. Needed to ensure feedback given to students is consistent and feasible. This is akin to a template.
  • Staff members should be able to establish a rubric for evaluating student performance in a discussion
  • Staff members should be able to use a grading system to assign marks to a student
  • Staff members should be able to request an admin-specific response. All requests should go to a list that staff and admin can see, and requests on the list are updated based on admin and staff feedback; requests can be closed and reopened, and when reopened, it links to the original request.

Now of course the above demo does not cover all of this, it moreso covers the student side posting and deleting process.
